# Kavo "Forest" Rill Blockout Submissions

**Room:** Meridian Prime Toy Development Room / Action Figure Prototype Lab  
**Purpose:** Drop zone for gray blockout screenshot packages  
**Canon Boundary:** Sandbox production folder. Does not alter main canon.

---

## Where To Drop the First Package

Create the first gray blockout submission package here:

`production/toy-prototypes/kavo-forest-rill/reference/blockout-submissions/kfr-blockout-v001/`

Do not put loose screenshot files directly next to this README.

---

## Required Folder Name Format

Use this format for every blockout submission:

`kfr-blockout-v###`

Examples:

- `kfr-blockout-v001`
- `kfr-blockout-v002`
- `kfr-blockout-v003`

Do not use vague names like `final`, `latest`, `newfinal`, `approvedmaybe`, or `use_this_one`.

---

## Required Protocol

Before submitting screenshots, follow:

`production/toy-prototypes/kavo-forest-rill/sculpt-notes/kavo-forest-rill-gray-blockout-screenshot-protocol.md`

Reviewers should evaluate the package using:

`production/toy-prototypes/kavo-forest-rill/sculpt-notes/kavo-forest-rill-gray-blockout-review-checklist.md`

---

## Minimum Required First Review Files

The first review package must include views 01 through 10:

1. front orthographic
2. side orthographic
3. back orthographic
4. three-quarter shelf read
5. head close-up
6. left wrist watch close-up
7. hands / grip close-up
8. boots / stance close-up
9. tail socket / balance close-up
10. jacket / harness / pants close-up

Also include:

- `kfr_blockout_v001_submission_notes.md`
- scene units: millimeters
- target height: 127 mm / 5 inches
- known unresolved issues

## Intake Rule

If a package is missing required views, uses inconsistent version numbers, or lacks submission notes, return it for cleanup before design review.

This folder is for blockout review packages only. Approved final reference assets should remain in the main `reference/` folder or their dedicated approval-note files.
