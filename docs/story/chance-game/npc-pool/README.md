# Meridian Prime NPC Pool

**Repository Status:** Sandbox / development only  
**Canon Status:** All entries are Non-Canon unless explicitly promoted after canon review.  
**Promotion Rule:** Requires user approval and canon review before any NPC may be treated as canon.

This folder stores sandbox NPCs for Meridian Prime chance stories, side-story seeds, comic one-shots, tabletop quests, game missions, anthology concepts, and prose drafts.

Core principle:

> The Story Game creates plot pressure. The NPC Pool creates people for the pressure to hit.

## Purpose

The NPC Pool is a reusable casting bench for the Meridian Prime Chance Engine.

Use it to:

- pull existing NPCs into chance stories;
- prevent every story from inventing disposable one-off characters;
- preserve roll receipts, canon risk, and story usefulness;
- track which NPCs are safe, restricted, recurring, or retired.

All generated NPCs remain:

```text
Sandbox NPC
Non-Canon
Requires Canon Review Before Promotion
```

## How the Story Game Room Should Use This Pool

1. Start with `npc-master-index.md` for fast casting.
2. Choose an NPC whose Story Use, Location, Faction, Tone, Pull Weight, and Canon Risk fit the scene.
3. Open the relevant pool file for the full NPC card.
4. Check `Do Not Use For` before using the NPC.
5. Generate a new NPC only when no existing one fits.

## Pool Placement Principle

An NPC’s primary pool is where their story pressure lives, not merely who signs their paperwork.

Example: an Antisapien witness attached to NCI through bad records belongs primarily in `antisapian-community-npcs.md`, with `nci` as a secondary tag.

## Saving Entries

Save each NPC into the most fitting pool file. Update `npc-master-index.md` when an NPC is added. Update `npc-tag-index.md` only when a new tag is introduced.

Where available, preserve roll receipts in the NPC card. Do not invent missing receipts.

## Canon Safety

NPCs may reference canon pressure, but they may not alter canon or solve protected mysteries.

A roll can point toward a protected mystery. It cannot answer it.

Do not use NPCs from this pool to solve the Core, define Dry Port truth, reveal Ildi’s restricted truth, reveal Rob Holliday’s full death-chain, identify the NCI assassin, or overwrite Book One.

## Promotion to Canon

Candidate Canon requires explicit user approval and canon review. Do not add sandbox NPCs to main canon indexes unless promotion is approved.

## Name Status

Generated names are sandbox generated, style-valid, not conlang canon, and require review before canon promotion.

## Final Principle

The pool organizes NPCs by story usefulness first, lore importance second.
