## Core Purpose

`Red Warrant` exists first as a music-forward game concept: a playable frame for generating killer 16-bit BGM for Devilment Studio.

All game lore, band lore, and artist lore should support soundtrack generation. The music is the main artifact.

---

## Core Premise

```text
The city did not fall. It was processed.
```

The player fights through a city where every district has been legally condemned while people are still living inside it. Entire neighborhoods are processed by civic machines, private enforcement crews, false courts, eviction engines, and document-fed violence.

The world is not post-apocalyptic. It is administrative apocalypse: everything has a stamp, a docket number, a red light, and a weapon.

---

## Gameplay Fantasy

```text
Fast side-scrolling action.
Heavy melee hits.
Short-range blast weapons.
Wall-smashing.
Boss arenas.
Corruption meters.
Ruined civic zones.
Brutal stage transitions.
```

The player should feel hunted by the city itself. Every stage is both a level and a legal sentence.

## Design Pillars

### 1. The City Is the Enemy

The setting should not feel like generic ruins. It should feel like municipal and corporate systems made physical.

Examples:

```text
permit offices as combat arenas
courtrooms as boss rooms
eviction machinery as stage hazards
black vans as enemy spawners
red stamps as execution marks
```

### 2. Every Level Has a Sentence

Each stage should feel like the player has been convicted before play begins.

### 3. Riffs Drive the Map

Track structure should suggest gameplay structure:

```text
intro pulse = stage entry
main riff = combat lane
breakdown = enemy wave / heavy encounter
lead motif = stage identity
final return = loop or exit pressure
```

### 4. Brutal But Playable

The BGM must be aggressive enough to sell the game, but not so busy that it fights gameplay feedback.
